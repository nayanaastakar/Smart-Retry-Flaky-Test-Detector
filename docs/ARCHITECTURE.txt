================================================================================
                         KEY ARCHITECTURAL PRINCIPLES
================================================================================

1. SEPARATION OF CONCERNS
   - Clear separation between presentation, business logic, and data layers
   - Page Object Model for test maintainability
   - Modular core services for reusability

2. CONFIGURATION-DRIVEN
   - Centralized configuration in config/config.py
   - Website-specific configurations in config/websites/
   - Easy to modify without code changes

3. AUTOMATED FAILURE HANDLING
   - Automatic screenshot capture on failures
   - Automatic console logs capture
   - Smart retry mechanism with exponential backoff
   - Flaky test detection and classification

4. AI-POWERED ANALYSIS
   - Detailed failure analysis using AI
   - Support for both cloud (OpenAI) and local (Ollama) AI
   - Structured prompts for consistent results
   - Integration in reports for actionable insights

5. COMPREHENSIVE REPORTING
   - Professional HTML reports with visualizations
   - Interactive charts using Chart.js
   - Progress bars for test distribution
   - Embedded screenshots and console logs
   - AI analysis display with structured formatting

6. MODERN WEB INTERFACE
   - Flask-based dashboard with Bootstrap 5
   - Responsive design for all devices
   - Real-time test execution feedback
   - Easy navigation between reports, logs, and screenshots

7. SCALABILITY
   - Configurable retry counts (1-20)
   - Support for multiple test scenarios
   - Easy to add new websites and tests
   - Modular architecture for extensions
